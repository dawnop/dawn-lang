# M6：博客后端重写 — 计划方案

> 状态：**已定稿**（2026-07-14，§8 四项决策已拍板）。
> 目标一句话：把 dawnop.com 的 FastAPI 后端整体替换为 Dawn 程序——前端零改动、
> 线上数据零迁移（共用同一 SQLite 文件与七牛空间）、nginx 按路由灰度切换、随时可回滚。
> 这是 design.md 排定的「真实生产 dogfood」：故意排在自举前，用真实项目把 stdlib
> 与报错的痛点在语言冻结前逼出来。

## 1. 迁移面盘点（现有后端全貌）

Python 侧 `backend/app`：FastAPI + SQLAlchemy + SQLite，8 张表
（users/articles/pages/tags/viz/file_objects/pending_uploads/settings）+
`article_fts` 虚表与三个同步触发器。约 55 个 REST 端点 + WebDAV 十余方法：

| 模块 | 端点 | 关键依赖 / 难点 |
|---|---|---|
| auth | 2 | bcrypt 校验、JWT HS256 签发（须与 PyJWT 互认，老 token 不失效）；login 是 **form-urlencoded** |
| articles | 9 | CRUD、分页、slug 生成、**multipart 导入 .md**、export 下载头（Content-Disposition） |
| pages | 8 | nav/排序/列表页文章分页 |
| tags | 7 | merge/cleanup、计数 |
| viz | 5 | 存前端编译产物，纯 CRUD |
| settings | 2 | key-value 与 DEFAULTS 合并 |
| search | 1 | **FTS5 simple→trigram→LIKE 三级降级** + 服务端高亮（转义后包 `<mark>`）。索引由 **SQL 触发器**在库内维护——写路径白捡，只需移植查询侧 |
| fm | 17 | 七牛 **HMAC-SHA1 签名**（上传凭证/私有下载 URL/管理 REST）、**302 重定向**、imageView2 缩略图、`?token=` 双通道鉴权、pending_uploads 孤儿账本；`/content`、`/upload` 是**二进制代理**（兜底路径） |
| monitor | 1 | 腾讯云 TC3-HMAC-SHA256 签名 + 七牛统计 + vault 探活（需 HTTP 客户端） |
| webdav | ~10 方法 | XML、Range 透传、流式 PUT、Basic auth + TTL 缓存——**二进制重灾区**，且已是独立子域名 `dav.dawnop.com` |
| 横切 | — | CORS、统一错误 `{"detail"}`、分页形状 `{total,page,size,items}`、启动 bootstrap（建触发器/FTS rebuild）、seed 脚本 |

## 2. 差距分析：Dawn 侧缺什么

已有的地基：playground `src/web`（路由 ADT + `{name}` 捕获 + 404/405 + 中间件组合 +
jdk.httpserver 每请求一虚拟线程）、纯 Dawn json 库（含 render）、互操作三件套（D8）。

硬约束（spec §9）与由此推出的缺口：

- **G1 外部 jar 无法进 classpath**。`dawn run/build` 是 `java -jar`，无 `--cp`；
  而 SQLite 驱动（sqlite-jdbc）不在 JDK 里，绕不开。design.md 本就预告
  「JWT/bcrypt/七牛用现成 Java 库」——前提就是这个。约束：Dawn 无依赖解析，
  外部 jar 限**单 jar、零传递依赖**（sqlite-jdbc、jBCrypt 满足；七牛官方 SDK
  拖 okhttp+gson 全家桶，**不用**，签名自己写）。
- **G2 无 SQL 能力**。`java.sql` 在 JDK 里，但 JDBC 全程抛 SQLException——Dawn
  无异常，**Java 异常过界即 panic**，一条坏 SQL 不能炸掉请求线程还泄漏连接
  （无 try/finally）。结论：stdlib `sql` 模块走 runtime ASM 类（Strings/Io/Maps
  先例），在 runtime 层捕获异常 → `Result`；事务给回调式 `with_tx(fn)`（runtime
  层包 commit/rollback）；内置单写连接串行化（虚拟线程并发写 SQLite 需单写者）。
- **G3 无加密原语**。七牛/腾讯云/JWT 全靠 HMAC。byte[] 可作局部变量但**不能进
  函数签名**（§9.5），纯 Dawn 拼 `Mac`+`Base64` 理论可行但每处都要 `String.new`
  中转、不可复用成函数。结论：stdlib `crypto`（hmac_sha1/sha256、sha1/sha256、
  base64/base64url 双向、随机 hex、常时比较——全部 JDK 内置算法，零外部依赖）。
- **G4 无 HTTP 客户端**。七牛管理 REST、register 的 stat 校验、monitor 三家探活
  都要发请求。`java.net.http` 的 `HttpResponse<T>.body()` 泛型擦除成不透明 Object，
  裸用体验差。结论：stdlib `http` 客户端薄包装（method/url/headers/body → Result）。
- **G5 web 框架缺半边**。Request **没有 headers**（Authorization 都读不到！）、
  Response 没有 headers（302 的 Location、下载的 Content-Disposition、401 的
  WWW-Authenticate 都写不出）；没有 form-urlencoded / multipart 解析；请求线程
  panic 未隔离成 500。结论：框架强化刀（详见 §5 刀 5）。
- **G6 二进制体**（后置）。Response body 是 String，`/fm/content`、`/upload` 代理
  与 WebDAV 都过不去。可行路径：byte[] 宽化为 `Object` 存进 Response 变体
  （`use java "java.lang.Object"` 可命名），server 侧原样写出。放最后，不阻塞主线。

## 3. 架构设计

- **进程形态**：单个 Dawn 程序监听 `127.0.0.1:8001`，systemd `dawnop-dawn.service`。
  先 **JVM 模式**跑（服务器已有 openjdk-21-jre-headless；`-Xmx` 设上限）；
  native-image 作 stretch——sqlite-jdbc 带 JNI 本地库，与 native-image 的兼容性
  未验证，不许诺（见 §7 风险）。
- **代码归属**（§8 决策 1）：业务代码放 **dawnop-site 仓库** `backend-dawn/`
  （与 nginx/systemd/部署脚本同仓，且开源仓库里躺一个 Dawn 实战后端本身就是展示）；
  `web`/`json` 库从 playground **vendored** 过去（先例：site/src/json 与 examples/m4/json
  双份同步）。stdlib 四个新模块（sql/crypto/http + 框架强化）落 dawn-lang 仓库。
- **模块划分**：`src/config`（.env 解析）、`src/db`（sql 之上的 repo 层，按资源分文件）、
  `src/api`（每资源一模块出路由表，main 拼装）、`src/qiniu`（签名 + 管理 REST）、
  `src/searchx`（降级阶梯 + 高亮）、`src/web` `src/json`（vendored）。无 ORM，
  手写参数化 SQL；无自动 OpenAPI（前端不依赖 /docs，接受）。
- **兼容性契约**：现有 JSON 响应**逐字段形状一致**是硬标准——分页形状、错误
  `{"detail"}`、datetime 的 ISO 格式（FastAPI 的毫秒/时区表示要对拍逐项核）、
  Int/Float 渲染。JWT 用同一 `SECRET_KEY` + HS256 + 相同 claims（sub/exp），
  老 token 无缝过渡。数据库直接打开生产同一文件，**不建表不迁移**（schema.sql +
  seed 脚本只服务全新安装）。
- **SQLite 双进程窗口**：灰度期 Python 与 Dawn 同时开库——WAL + busy_timeout，
  且纪律为「**Dawn 只读先行**，写路径整模块原子切换」，避免双写者。切换完成后
  Dawn 是唯一写者（sql 模块内单写连接）。

## 4. 迁移与部署策略（绞杀者模式）

nginx 是现成的流量刀：`/api/` 下按 location 前缀逐步把路由从 `:8000`（uvicorn）
指到 `:8001`（Dawn），每步一行 `proxy_pass`，回滚 = 改回来 reload，秒级。

顺序：`/api/health`（冒烟）→ 公开只读（articles/pages/tags/viz GET + search，
共库只读，最安全）→ auth + 后台写（按资源整模块切，杜绝同表双写者）→ fm →
monitor → 全量。uvicorn 退役后仅剩 `dav.dawnop.com` 反代（若 WebDAV 留 Python）。

**对拍**是主测试手段：契约快照矩阵（curl 请求集 → 双后端 → jq 规范化 → diff），
进 `test/contract.sh`（playground 先例）；纯逻辑（slug/分页/高亮/签名）用 Dawn
`test` 块；pytest 全程保留当规格参照，直至 Python 退役。

## 5. 刀法（每刀一 commit，全量测试绿再进下一刀）

**预备五刀（dawn-lang 仓库，语言/stdlib/框架）**

1. **CLI classpath**：`dawn run/test/build --cp <jars>`（运行期拼 `java -cp app.jar:...`
   或 manifest Class-Path；`use java` 到第三方类时编译器反射 classpath 同步吃到）。
   spec §12 CLI 章配套。
2. **stdlib `sql`**（本刀先做 **spike**：sqlite-jdbc 打开生产库副本 + FTS5 可用性 +
   `load_extension` 加载 libsimple——通不过则搜索按 §7 降级预案走）：open/query/exec
   参数化、行取值 API、`with_tx`、异常全 Result 化、单写者串行化。
3. **stdlib `crypto`**：hmac_sha1/hmac_sha256/sha1/sha256、base64/base64url 编解码、
   random_hex、constant_time_eq。
4. **stdlib `http` 客户端**：request(method, url, headers, body) → Result[Resp]，
   超时可配。
5. **web 框架强化**：Request.headers、Response.headers + redirect/attachment 助手、
   form-urlencoded 与 multipart（先文本件）解析、每请求 panic→500 隔离、
   CORS 中间件。playground runner 同步迁移受益，作回归场。

**业务八刀（dawnop-site 仓库）**

6. **骨架**：config/.env、db 连接、repo 层起步、`/api/health`、统一错误形状、
   CORS/日志中间件、systemd 单元 → **部署冒烟**（nginx 只切 health）。
7. **auth**：bcrypt 校验（方案见 §8-3）+ JWT 签发/校验互认 + 鉴权中间件
   （含 `?token=` 双通道）。
8. **公开只读全量**：articles/pages/tags/viz 公开 GET + search 查询阶梯与高亮
   → 契约对拍 → **灰度第一单元上线**。
9. **后台写**：articles/pages/tags/settings CRUD、slug 生成、reorder、
   import（multipart）/export（下载头）。
10. **viz 写 + 搜索收尾**：viz CRUD、simple/trigram/LIKE 全阶梯线上验证。
11. **fm 全量**（最大一刀，必要时再拆二）：路径树 CRUD、七牛签名（凭证/私有 URL/
    管理 REST：stat/delete/copy/move）、302 预览/下载、imageView2 缩略图、
    upload-token/register/账本、stats、search。`/content` `/upload` 二进制代理暂留
    Python（nginx 这两条路由先不切）。
12. **二进制体 + monitor**：G6 落地 → `/content` 收编（响应二进制体，字节精确）；
    腾讯云 TC3 签名（vs SDK 逐字节）+ monitor 四块聚合（server /proc、lighthouse
    traffic/instance/cpu_trend、qiniu kodo+CDN+respack、vault 探活）+ TTL 缓存
    （120s/300s/600s）+ fm/stats respack 配额，与 FastAPI 逐字段对拍一致。
    附带修 JSON 解析器整数保真（`JInt`，避免 cache round-trip 把字节数变成 `x.0`）。
    **`/upload`（后端代理上传）改随 WebDAV 一并推迟 M6.5**：它是 multipart **二进制
    请求体**，与 WebDAV PUT 同缺"web 框架二进制请求体"能力（当前 `read_body` 按
    UTF-8 解码会毁坏二进制），且非前端主路径（前端走 upload-token+直传+register）。
    切流时 `/api/fm/upload` 与 `/dav` 一样留 Python。
13. **切流收尾**：nginx 全量切换、退役 uvicorn（留 dav）、seed/运维脚本、
    docs 更新（spec 新 stdlib 章、design.md 里程碑记录、教程若干节）、进展文章。
    注意：本文件与新 stdlib 文档入站点时要登记 `site/src/gen/common.dawn` 的
    `map_doc_url`（链接 lint 会 panic）。

**M6.5：WebDAV in Dawn + fm `/upload` 代理**（§8 决策 2，已列入）：XML/PROPFIND、
Range 透传、流式 PUT、Basic+TTL 缓存——共同前置是"web 框架二进制请求体"（把
`getRequestBody` 的 byte[] 以 opaque `Object` 交给 handler，multipart-over-bytes
解析）。独立子域名/备份路径使其不阻塞 M6 验收，M6 收尾后单独立项。

估算：预备刀各 0.5–1 个会话；业务刀里 fm 最大（≈2 刀量），其余各 ≈1；
全程约 12–15 个 commit，与 M4/M5 体量相当偏大。

## 6. 验收标准

- **前端一行不改**，全功能通过：公开站（列表/文章/搜索 ⌘K/viz 交互图）+ 后台
  （文章/页面/标签/设置/viz/文件管理器含直传与缩略图/监控页）。
- 契约对拍矩阵全绿；`dawn test` 全绿；pytest 在参照期同绿。
- 生产全量切换后观察 7 天（journalctl + monitor 页）：无异常 5xx、内存稳定在
  `-Xmx` 内、老 JWT 无缝。
- 语言侧产出清点：sql/crypto/http/框架强化进 spec + `dawn doc` + 教程；
  痛点清单（本里程碑的真正目的）归档进 design.md。

## 7. 风险与对策

| 风险 | 对策 |
|---|---|
| sqlite-jdbc 加载 libsimple 扩展失败（JNI 环境差异） | 刀 2 先 spike；失败则搜索走 trigram/LIKE（Python 侧同款降级阶梯，本就设计为可退化） |
| Java 异常过界 panic | sql/http 全在 runtime 层捕获转 Result；`use java` 直调（如 jBCrypt）外包一层防御；框架每请求 panic 隔离（刀 5） |
| JSON 细节形状差异（datetime 毫秒/时区、浮点渲染、null 字段有无） | 对拍矩阵先于切流；发现一处修一处，契约以 Python 现状为准 |
| SQLite 双进程写冲突 | 灰度纪律：Dawn 只读先行，写路径按资源整模块原子切换，任何表不出现双写者 |
| 4G 服务器再加常驻 JVM（已有 vault/dawn-play/uvicorn） | `-Xmx` 上限 + monitor 页观察；uvicorn 退役后净增有限；native-image 作远期减负 |
| native-image × sqlite-jdbc（JNI + 反射） | 不入验收标准，stretch；JVM 模式即达标 |
| multipart/二进制被 String 框架破坏 | 二进制路由在 G6 落地前一律不切给 Dawn |
| 外部 jar 无依赖解析 | 白名单制：仅单 jar 零传递依赖（sqlite-jdbc、jBCrypt），其余一律手写（七牛签名/JWT） |

## 8. 决策记录（2026-07-14 已定）

1. **业务代码归属** = dawnop-site 仓库 `backend-dawn/`（部署一体、开源展示）。
2. **WebDAV** = 列入 **M6.5**（M6 期间留 Python 常驻，M6 验收后单独立项迁 Dawn）。
3. **密码哈希** = **jBCrypt** 单 jar（现有 bcrypt 哈希零迁移，老密码直接可用）。
4. **契约严格度** = JSON 响应形状严格对齐；422/400 等边角状态码允许合理化，
   凡有出入逐条记录在对拍清单里。
